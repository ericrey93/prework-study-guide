HTML

  • The head element contains information about the webpage.
  • The body elements represent the visible content shown to the user
  • semantic HTML elements is the name of the element that describes its content and the role it plays on a webpage
  • when coding an image, be sure to use src for the link of the image and alt to describe the image
  • using anchor, or a, creates links to other webpages, then use href to point to the URL for that link

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • you need to link your external file to the main file to make changes
  • the rel specifies the relationship between the current document and the linked document

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add[file]: adds a file as it looks to your next commit
  • git pull origin main: good habit to pull main branch into the feature branch
  • git push origin 'branch name': this will push up all changes
  • git reset[file]: unstage a file while retaining the changes in working directory
  • git diff: difference of what is changed but not staged
  • git commit -m"[descriptive message]": commit your staged content as a new commit snapshot
  • git branch: will list all your branches and a * will appear next to your active branch
  • git merge[branch]: merge the specified branch's history into the current one
  • git log: show the commit history for the current active branch

Javascript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Conditional statements and loops can be used to control the order in which the code is executed
  • we can use arrays to hold a group of data that is related in some way
  • In an array, everything can be located through its index number, and indices always starts a 0
  • loops must always have a starting point, a condition, and a final statement that allows the array to iterate over each item
  • you can comment out code by using /* before the code starts and */ when the code ends. this will deactivate the code without deleting it
  • functions are a simple set of instructions that tells the computer how to perform a certain task
  • functions do not automatically execute, and must be called upon to be used
